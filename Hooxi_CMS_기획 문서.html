좌측 고정 메뉴(Left LNB) 기반의 웹페이지 메뉴 구조(IA)를 핵심 CRM 기능 

 

 

 

 

1. 대시보드 

시스템 접속 시 노출되는 메인 화면으로, 핵심 지표와 개인별 당일 업무 현황 파악. 

 

● 통합 현황 : 등록된 운수사 및 건물 고객사 총계, 연동된 차량 및 고정원 설비 규모 

● 나의 업무 : 오늘 예정된 고객사 미팅, 신규 접수된 할당 이슈, 미회신 카카오톡/이메일 건수 

● 시스템 상태 : 전일 심야 및 당일 정오 기준 데이터 백업 정상 수행 여부, AI 서버 상태 

 

2. 고객사 마스터 관리 

고객 정보와 설비 제원을 중앙화하고, 조건에 맞게 필터링 조회. 

 

가. 전체 고객사 목록 : 모든 고객사의 기본 정보, 상태, 담당자 연락처를 그리드 형태로 제공 

나. 모빌리티 뷰 : 운수사 전용 목록으로 차량 대수, 내연기관 및 전기차 비율, FMS 연동 상태 중심 조회 

다. 고정원 뷰 : 건물 및 농장 전용 목록으로 태양광 용량, 히트펌프 대수 중심 조회 

라. 고객사 상세 페이지 : 개별 고객사의 기본 정보 및 활동 이력 타임라인(통화, 미팅 기록) 통합 열람 

 

3. 커뮤니케이션 및 이슈 관리 

고객 대응 창구를 단일화하고 이슈의 처리 상태 추적. 

 

마. 통합 이슈 관리 : 접수, 원인 파악, 처리 중, 처리 완료 상태별 이슈 목록 및 담당자 배정 현황 

바. 카카오톡 대응 채널 : 고객사가 카카오톡으로 발송한 문의 내역 조회 및 1:1 답변 처리 공간 

사. AI 메일 스테이션 : 수신된 외부 이메일 열람 및 AI 답장 초안 생성 버튼을 통한 빠른 회신 업무 처리 

 

4. 문서 및 자산 관리 

고객사별 계약서 및 업무에 필요한 모든 증빙 서류를 체계적 보관. 

아. 고객사 전용 문서함 : 마스터 데이터와 연동되어 자동 생성되는 고객사별 클라우드 폴더 

자. 표준 양식 보관소 : 내부 표준 계약서, 개인정보 활용 동의서 등 공통 문서 템플릿 다운로드 센터 

 

5. 시스템 환경 설정 

관리자 권한으로 시스템의 접근 권한과 유지보수 환경 제어. 

 

● 계정 및 권한 관리 : 내부 직원 계정 생성 및 직급별 메뉴 접근 권한 통제 

● 데이터 백업 센터 : 일 2회 자동 스냅샷 이력 조회 및 문제 발생 시 특정 시점 복구 실행 

● AI 지식 관리 : AI가 답변을 생성할 때 참조하는 내부 규정 문서 및 FAQ 데이터베이스 업데이트 

 

 

DB 기초 설계 

 

향후 확장성과 성능을 고려하여 고객사 기본 마스터 테이블과 도메인별 자산 테이블을 분리(정규화), 모든 이력을 추적할 수 있도록 설계. 

 

1. 데이터베이스 설계 원칙 

■ 식별자(PK) 체계
모든 테이블의 기본키는 고유 문자열(UUID 또는 규칙 기반 문자열)을 사용하여 향후 데이터 이관 및 확장 대비함. 

■ 외래키(FK) 무결성
모든 활동 이력과 자산 정보는 고객사 식별자를 참조하여 데이터 고아가 발생하지 않도록 통제. 

■ 이력 추적
모든 데이터의 등록일시(created_at)와 수정일시(updated_at)를 기록하여 감사(Audit) 요건 충족.

UI/UX 화면 설계 

 

1. UI/UX 디자인 원칙 및 디바이스별 대응 전략 

● 시각적 안정성
장시간 수많은 데이터를 다루는 실무진의 눈 피로도를 낮추기 위해, 전자잉크(E-ink) 디스플레이나 종이 문서를 보는 듯한 무채색 기반의 미니멀 테마 적용 

 

● 모바일 뷰어 모드
현장 이동 및 전화 통화 중 빠른 조회를 위한 화면. 복잡한 데이터 표는 직관적인 카드 형태로 자동 변환되며, 주 담당자 연락처 터치 시 즉시 전화가 걸리는 통화 연동 기능 지원. 데이터 등록 및 복잡한 수정 기능은 오작동 방지를 위해 숨김 처리 

 

● 태블릿 현장 지원 모드
농장 및 차고지 현장 실사 지원. 터치하기 편하도록 여백을 충분히 확보하고, 태블릿 카메라를 연동하여 태양광 인버터나 히트펌프 제원표를 촬영 후 즉시 시스템에 업로드하는 기능 활성화 

 

● 데스크톱 풀 스케일 모드
메인 업무용 컨트롤 타워. 마스터 데이터 관리, AI 이메일 초안 수정, 대량 엑셀 다운로드 등 시스템의 모든 권한과 기능이 제한 없이 동작 

 

2. 공통 레이아웃 (데스크톱 기준) 

● 최상단 헤더
중앙에 고객사명, 사업자번호, 연락처를 모두 찾을 수 있는 통합 검색창 배치. 우측에는 AI 채팅 담당자 이관 요청 등을 알리는 알림 센터와 현재 접속 중인 담당자 프로필 표시 

 

● 좌측 고정 메뉴 (Left LNB)
대시보드, 고객사 마스터, 커뮤니케이션, 문서 자산, 환경 설정으로 구성. 넓은 데이터 조회가 필요할 때는 텍스트가 사라지고 아이콘만 남는 화면 확장 기능 제공 

 

3. 주요 화면별 기능 정의 

● 화면 1. 대시보드 (통합 현황판) 

● 자산 및 재무 성과
총 연동 차량 대수, 고정원 설비 규모, 그리고 10~15% 성공 보수 모델 기반의 당월 예상 탄소 배출권 정산 금액을 상단 위젯으로 표출
 

● 시스템 안전 상태
일 2회(정오, 자정) 자동으로 수행되는 데이터베이스 백업의 정상 완료 여부를 시각적으로 표시
 

● 전사 업무 보드
팀 전체 공유 원칙에 따라, 금일 예정된 모든 고객사 미팅 일정과 현재 접수 및 처리 중인 이슈 리스트를 종합하여 표시 

 

● 화면 2. 고객사 마스터 목록 (데이터 그리드) 

● 분류 탭
전체 리스트, 운수사(모빌리티) 전용, 건물 및 농장(고정원) 전용으로 탭 분리
 

● 맞춤형 컬럼
운수사 탭에서는 내연기관 및 전기차 대수와 FMS 연동 여부를 중점적으로 출력하고, 고정원 탭에서는 발전 용량 및 설비 대수를 중점적으로 출력
 

● 빠른 검색
각 컬럼의 헤더를 클릭하여 오름차순/내림차순 정렬 기능을 제공하며, 항목 클릭 시 상세 화면으로 이동 

 

● 화면 3. 고객사 마스터 상세 (이력 타임라인) 

● 좌측 정적 패널
법인 상호, 사업자번호, 주 담당자 연락처 등 기본 정보와 해당 고객사의 계약 상태 배치
 

● 우측 동적 타임라인
전화 통화, 미팅, AI 챗봇 발화 내역 등 모든 상호작용 기록이 시간 역순으로 스크롤됨. 타 담당자가 응대하고 남긴 기록도 모두 실명과 함께 노출되어 팀 내 완벽한 맥락 공유 지원 

 

● 화면 4. 커뮤니케이션 및 하이브리드 대화 관제 

● 채팅방 리스트
활성화된 카카오톡 상담 목록을 좌측에 배치하고, 상태를 AI 응대 중, 직원 이관 대기, 담당자 응대 중으로 구분
 

● 실시간 대화창
중앙 화면에 AI와 고객의 대화 내역 전체 출력. 직원이 텍스트 입력창에 커서를 두고 타이핑을 시작하면, 즉시 담당자 응대 모드로 자동 전환되며 AI 개입 차단
 

● 고객 상황 패널
대화창 우측에 현재 상담 중인 고객사의 자산 규모와 과거 이슈 리스트를 띄워, 화면 이동 없이 정확한 응대 지원 

 

● 화면 5. 문서 및 자산 아카이브 

● 폴더 탐색기
좌측에 표준 양식(KOC, VCM 방법론 가이드 등) 공용 폴더와 고객사 상호명으로 자동 생성된 아카이브 폴더 트리 제공
 

● 파일 리스트
우측에 선택한 폴더의 파일 목록 표출. 드래그 앤 드롭 업로드를 지원하며, 삭제 시 즉각 반영되지 않고 휴지통으로 이동 후 팀장 승인을 거치도록 설계 

 

4. 모바일 (스마트폰) : 빠른 조회 및 현장 대응 (View & Call) 

운전 중이거나 외부 이동 중 전화 통화를 하면서 필요한 정보를 즉시 찾아보는 '뷰어(Viewer)' 역할에 집중 

 

● 화면 레이아웃 변경 

● 좌측 고정 메뉴(Left LNB)는 숨김 처리하고, 우측 상단 햄버거 버튼(三) 또는 하단 탭 바(Bottom Navigation) 형태로 메뉴 구조 단순화
 

● 복잡한 데이터 그리드(표)는 가로 스크롤을 유발하므로, 고객사명과 핵심 지표만 크게 보이는 '카드(Card) 뷰' 형태로 자동 변환되도록 설계 

 

● 핵심 지원 기능 

● Click-to-Call 연동
고객사 마스터 정보에서 주 담당자 전화번호를 터치하면, 즉시 스마트폰의 전화 앱으로 연결되도록 지원 

● 통합 검색 최상단 고정
화면을 내리더라도 검색창이 상단에 고정되어, 통화 중에도 즉시 다른 고객사의 이름으로 검색 

● 알림 및 상태 확인
현재 진행 중인 카카오톡 문의나 긴급 이슈 상태만 빠르게 확인 

 

● 기능 제한 (입력 저항 최소화) 

● 신규 고객사 등록, 복잡한 엑셀 데이터 다운로드, AI 이메일 초안 생성 및 장문 수정 기능 등은 모바일 화면에서 숨김(Hidden) 처리하여 오작동 방지. 

 

5. 태블릿 (Tablet) : 현장 실사 및 미팅 지원 (Touch & Work) 

농장이나 운수사 차고지 현장 실사, 또는 외부 미팅 시 문서를 확인하고 즉각적인 현장 데이터를 입력하는 '포터블 워크스테이션' 역할 수행 

 

● 화면 레이아웃 변경 

● PC와 유사한 화면 구성을 유지하되, 마우스가 아닌 손가락 터치를 고려하여 버튼과 입력창의 상하 여백(Touch Target)을 넉넉하게 확보
 

● 태블릿 가로 모드(Landscape)에서는 좌측 메뉴와 우측 상세 화면을 분할(Split)하여 PC처럼 사용하고, 세로 모드(Portrait)에서는 좌측 메뉴를 숨겨 화면 공간 확보 

 

● 핵심 지원 기능 

 

● 카메라 연동 현장 첨부
태블릿으로 태양광 인버터, 히트펌프 제원표, 차량 계기판 등을 촬영하여 '고객사 전용 문서함'에 즉시 업로드하는 기능 활성화
 

● 전자 서명 및 문서 열람
다운로드 센터에 있는 표준 양식(KOC/VCM 계약서 등)을 열람하고, 현장 미팅 시 화면에 직접 서명을 받을 수 있는 인터페이스 고려
 

● 카카오톡 하이브리드 대응
화면 분할을 통해 좌측에서는 고객사 자산 현황을 보고, 우측에서는 카카오톡 실시간 답변을 입력하는 멀티태스킹 지원 

 

6. 데스크톱 (PC) : 풀 스케일 업무 및 관리 (Heavy Work) 

내부 사무실에서 대량의 데이터를 다루고 문서를 편집하는 '메인 컨트롤 타워' 역할 수행
앞서 기획한 종이 문서나 전자잉크를 보는 듯한 미니멀하고 눈이 편안한 UI가 가장 잘 발휘되는 환경 

 

● 화면 레이아웃 및 기능 

● 앞서 정의한 좌측 고정형(Left LNB) 메뉴와 다중 분할 패널을 100% 활용
 

● 모든 등록, 수정, 삭제, 엑셀 업로드/다운로드, 시스템 백업 관리, AI 프롬프트 지식베이스 관리 등 관리자 및 실무자의 모든 권한이 제한 없이 동작 

 

7. 민감 정보 마스킹 적용 대상 

시스템 내에서 기본적으로 가려져야 할 재무 및 계약 관련 데이터의 범위 정의 

 

● 수익 및 정산 지표 : 대시보드의 당월 예상 정산액, 누적 매출, 성공 보수 금액 

● 계약 조건 : 고객사 마스터 상세 화면의 성공 보수율(예: 15%), 수수료, 단가 

● 민감 식별 정보 : 필요한 경우, 전체 고객사 목록에서 타 고객사의 연락처나 세부 자산 규모 

 

8. UI/UX 동작 방식 (Click-to-Reveal) 

사용자의 불편을 최소화하면서 보안을 유지하는 3단계 상호작용 설계 

 

● 기본 상태 (Default Masking) 

● 시스템 로그인 시 모든 민감 정보는 기본적으로 블러(Blur) 처리되거나 별표(예: %) 기호로 치환되어 렌더링 

 

● 글로벌 보안 토글 (Global Toggle) 

● 상단 헤더(Top Header) 알림 센터 옆에 '보안 모드' ON/OFF 스위치 배치 

● 현장 미팅 직전, 이 스위치를 통해 시스템 전체의 마스킹 상태를 일괄적으로 켜거나 끌 수 있음 

 

● 개별 클릭 표시 및 자동 숨김 (Micro-Interaction) 

● 보안 모드가 켜진 상태에서 특정 숫자를 확인해야 할 경우, 마스킹된 텍스트 영역을 클릭하면 해당 정보만 일시적으로 표시 

● 보안 사고 방지를 위해 표시된 정보는 마우스 커서가 영역을 벗어나거나(Mouse-leave), 5초가 경과하면 자동으로 다시 가려짐(Auto-hide) 

 

9. 화면 스토리보드 업데이트 반영사항 

앞서 정의한 주요 화면들에 마스킹 정책 적용 

 

● 대시보드 (통합 현황판)
당월 예상 정산액 위젯의 타이틀(성공보수 %)과 메인 금액(₩ ,,*)이 블러 처리 

● 고객사 마스터 상세 (우측 패널)
고객 정보 요약 영역에 위치한 '성공 보수율' 항목의 수치가 마스킹 

● 커뮤니케이션 (하이브리드 챗)
AI 챗봇이 과거 대화 내역에서 정산 금액을 언급한 부분이 있다면, 화면에 텍스트를 뿌려줄 때 해당 금액 부분만 정규식을 통해 자동 마스킹 처리되도록 프론트엔드 로직 추가
